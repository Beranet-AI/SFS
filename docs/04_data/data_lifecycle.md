# Data Lifecycle

This document describes how data flows and evolves in SFS.

---

## Telemetry Lifecycle

1. Generated by device
2. Forwarded by edge node
3. Ingested by Data Ingestion service
4. Persisted by Management
5. Evaluated by rules and AI
6. Reflected in live status
7. Archived or expired

Telemetry is immutable once stored.

---

## Health Data Lifecycle

1. Telemetry batch evaluated
2. Health record created
3. Historical trend preserved
4. May trigger incident
5. Used for AI retraining

---

## Incident Lifecycle

1. Created (rule / AI / manual)
2. Open
3. Acknowledged
4. Resolved
5. Archived

All state transitions are logged.

---

## LiveStatus Lifecycle

- Built from latest telemetry
- Cached in memory or Redis
- Reconstructed on restart
- Never persisted as business data
